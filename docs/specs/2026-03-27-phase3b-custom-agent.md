## 1. 概要

ユーザーが自由にエージェントを定義・追加できる UI を実装する。
プリセットテンプレートの選択とゼロからのカスタム作成の両方をサポート。

## 2. スコープ

| # | 機能 | 詳細 |
| --- | ------ | ------ |
| 1 | エージェント作成モーダル | 名前・アバター・ロール・カラー入力フォーム |
| 2 | Big Five スライダー | 5つの性格パラメータを視覚的に調整 |
| 3 | プリセットテンプレート | Kai/Mia/Rex + Nova/Echo/Ash |
| 4 | カスタムボイススタイル | 口調・一人称・語尾のカスタマイズ |
| 5 | エージェント削除 | 確認ダイアログ付き削除機能 |
| 6 | ワールド作成時のエージェント選択 | デフォルト3体 or カスタム選択 |

### CROSS-AI レビュー結果（2026-03-27 確定）

**1. UI状態管理** → `formState` オブジェクト + `applyStateToDOM()` + `buildAgentData()` + `validateFormState()`
- プリセット選択時は `formState` を一括更新し、`applyStateToDOM()` でスライダー位置・値表示を同期
- バニラ JS に双方向バインディングはないため、手動同期パターンを採用

**2. 絵文字ピッカー** → カスタムグリッド + 「その他入力」フリーテキストフィールド
- Unicode 13 以前の1コードポイント絵文字をグリッド表示
- 「その他を入力」フィールドで任意の絵文字を直接入力可能（フォールバック経路）
- `Intl.Segmenter` で1セグメント＝1表示文字を保証するバリデーション

**3. MessageBus 動的統合** → 4点の改善
- ✅ `handleSend` に `state.isTyping` ガード追加（**Task 0 で修正済み**）
- `onSnapshot` 内でエージェント差分を検知し、ハートビートの開始/停止を制御
- `createAgent` にエージェント上限チェック（6体）を追加
- `channel.members` の不整合は TODO コメントで対応（Phase 4 で解決）

### スコープ外

- Cloud Functions 移行（Phase 4 で対応）
- 自動 E2E テスト（Playwright）導入
- エージェント間の「チーム」概念
- エージェント画像アバターのアップロード

## 8. リスクと軽減策

| リスク | 影響 | 軽減策 |
| -------- | ------ | -------- |
| エージェント数増加でハートビート API 負荷増 | 429エラー | エージェント上限 6体、ハートビート間隔調整 |
| MessageBus への動的追加で購読漏れ | メッセージ受信失敗 | 追加時に明示的 subscribe + 既存チャネルへの通知 |
| 絵文字ピッカーがモバイルで使いにくい | UX低下 | 「その他を入力」フリーテキストフィールドで直接入力可能 |
| handleSend の二重発火 | エージェントが重複応答 | `state.isTyping` ガードで防止（Task 0 で修正済み） |
